# 一手信息源清单：财报 / 招股书 / 访谈 / 播客

> 最后更新：2026-04-22

## 摘要
行业研究的质量由**信息源的质量**决定。本文把常用信息源按**信号强度**排成"金字塔"，给出每一类的获取方式和坑。**底线：永远要求至少两个独立一手源互相印证。**

## I. 法定披露（最强）

### 美股公司
- **SEC EDGAR**（<https://www.sec.gov/edgar>）——所有 10-K / 10-Q / 8-K / DEF 14A / S-1 原始档案；关键字搜索 ticker 或 CIK
- **Investor Relations 页**——每家公司官网都有，常包含季报 earnings slides + webcast 录音
- **Seeking Alpha 的 earnings call transcripts**——免费看，虽然偶尔延迟

### 港股 / A 股
- **港交所披露易**（<https://www.hkexnews.hk>）——招股书 / 年报 / 自愿公告
- **巨潮资讯网**（<http://www.cninfo.com.cn>）——深交所上交所官方披露
- **东方财富 / 同花顺的 F10**——快速看财务摘要

### 一级市场 / 未上市
- **Crunchbase**（付费但试用版够用）——融资历史、投资方
- **PitchBook / CB Insights** ——更深度但贵，很多学校和大公司有账号
- **Dealroom**、**IT 桔子**（中文）——替代品
- **SEC Form D**——私募融资的美国合规文件，可以挖出未公开轮次

### 读财报的重点顺序
1. **Management's Discussion & Analysis (MD&A)**——管理层自己对业务的解读
2. **Risk Factors**——监管、业务、技术、供应链的自认风险
3. **Segment reporting**——按业务线拆收入
4. **Subsequent events**——财报发布后到披露前的重大事件

!!! tip "不要只看利润表"
    现金流量表比利润表更难粉饰。AI 公司尤其要看 **deferred revenue**（预收款）、**customer concentration**（客户集中度）、**capex on H100/H200** 这种大额资本支出。

## II. 当事人长访谈（1h+）

**规律**：15 分钟的财经访谈信息密度≈推特短文；60 分钟深访才值得听；90 分钟 + 的对谈是金矿。

### 推荐的访谈源（英文）
| 节目 | 特色 | 嘉宾画像 |
|---|---|---|
| **Dwarkesh Podcast** | AI 技术深度访谈，追问精准 | Dario Amodei、Demis Hassabis、Jeff Dean |
| **Latent Space** | AI 工程/产品一线 | 各家 CTO / VP、开源项目主理人 |
| **Acquired** | 讲公司史讲得最好 | 每期一家公司 3-5 小时 |
| **Invest Like The Best** | 投资人视角 | CEO、基金经理 |
| **The Logan Bartlett Show** | VC 视角 | SaaS/AI 创始人 |
| **20VC** | 超高频，风投圈 | 各路 GP |
| **Lenny's Podcast** | 产品视角 | 头部产品负责人 |
| **Stratechery Interviews (Ben Thompson)** | 商业模式分析 | 平台型公司 CEO |

### 中文访谈源
| 节目 | 特色 |
|---|---|
| **张小珺 商业访谈录** | 国内一线公司 CEO / CTO，深度可读 |
| **晚点聊 LateTalk** | 晚点主笔做的长播客 |
| **十字路口** | AI 产品经理视角 |
| **科技早知道（小宇宙）** | 头条科技话题，偏新闻解读 |
| **声东击西** | 宏观科技/人物 |

### 官方发布会 / 年度大会
- OpenAI DevDay、Anthropic 的 Opus 发布、Google I/O、Meta Connect、NVIDIA GTC、Apple WWDC、字节火山引擎大会、阿里云栖大会——**所有重要产品信息都先出现在这里**
- 完整 keynote 录像 + 配套技术博客比新闻报道信息量大 10 倍

### 访谈的识别信号
- **有没有"新信息"**——嘉宾说了什么是**之前没说过**的
- **主持人敢不敢追问**——软性 PR 性质的访谈价值低
- **嘉宾是不是真的 decision maker**——VP 级比总监级有用得多

- **OpenAI Blog**: <https://openai.com/blog>
- **Google DeepMind Blog**: <https://deepmind.google/discover/blog>
- **Meta AI Blog**: <https://ai.meta.com/blog>
- **NVIDIA Technical Blog**: <https://developer.nvidia.com/blog>
- **HuggingFace Blog**: <https://huggingface.co/blog>
- **字节 Seed / ByteDance Research**
- **DeepSeek 技术博客**
- **Physical Intelligence Blog**

### 论文与预印本
- **arXiv** → cs.AI / cs.CL / cs.LG / cs.RO → 订阅 RSS
- **Papers With Code**——带代码的论文
- **HuggingFace Daily Papers**——每日精选
- **Semantic Scholar**——找文献引用关系

### GitHub 活动
一家 AI 公司的 **真实投入** 往往体现在 commit 频率、活跃 contributor、issue 响应速度上。PR 文案可以包装，但 GitHub commit log 骗不了人。

- **Import AI**（Jack Clark，Anthropic 联合创始人）——每周 AI 总结
- **Ahead of AI**（Sebastian Raschka）——ML 技术
- **AI Snake Oil**（Arvind Narayanan）——反炒作
- **Interconnects**（Nathan Lambert）——后训练与开源
- **The Algorithmic Bridge**（Alberto Romero）——Big Picture

**推特 / X**（关注列表推荐）
- 各家公司官方号
- 公司 VP 以上管理层（个人号比公司号有料）
- 独立研究员：@karpathy、@DrJimFan、@soumithchintala
- 记者：@steph_palazzolo (The Information)、@emilymbender、@michael_nielsen

**小红书 / 知乎**（中文一线从业者）

| **Bloomberg** | 财经 + 科技双强 | ~$350 |

## VI. 中文财经 / 科技媒体（交叉验证用）

- **晚点 LatePost**（36Kr 旗下）——深度专题好
- **36Kr**、**极客公园**、**PingWest 品玩**——新闻速度快
- **硅星 GenAI**、**机器之心**——技术 + 行业
- **虎嗅**、**钛媒体**——观点类，需要筛选

**警告**：大部分中文科技媒体的"独家访谈"有软文成分，用它们补事实而不是观点。

## VII. 自媒体 / 公众号（谨慎使用）

- **公众号 AI 头部号** 有几个值得订阅（如"量子位"、"新智元"、"AI 前线"），但它们的内容多半是编译自英文源——**直接读英文一手源更高效**
- **B 站 / 抖音科技博主**——极少数有深度，主要是传播价值
- **朋友圈转发的"独家内幕"**——几乎都是错的，除非出处明确

## VIII. 不用的（坚决避开）

- 只有"据悉" / "有消息称" / "据知情人士"且无具名的爆料（有具名 = 强信号）
- 没有数据来源的图表
- 翻译自推特短帖的长文包装
- 带强烈情绪（"颠覆！"、"崩盘！"、"碾压！"）的文章
- 百度百科（版本混乱）、Wikipedia 中文版（经常滞后）

## 信息源维护工作流

**订阅**：
1. RSS 工具：**Inoreader** / **Feedly**
2. Newsletter 统一入口：用一个专门邮箱（避免污染日常邮箱）
3. 播客：**小宇宙**（中文）+ **Overcast / Pocket Casts**（英文）

**整理**：
1. 一个 Obsidian / Notion 库作为"链接抽屉"，按公司 / 话题 tag
2. 每周清一次——未读堆太多要砍
3. 每月选 3-5 篇真正读完 + 写 150 字摘要——这是**内化**的关键

## 常见错误

1. **信源广度陷阱**：订了 100 个 newsletter 但一个都没读完——**5 个订阅读完比 100 个订了不读强**
2. **二手复述循环**：A 媒体引 B 媒体引 C 报告引 D 调研——追溯到 D 才算到一手
3. **时间盲区**：忘记标注信息时点，把 3 年前的行业格局当成现在
4. **中英文偏科**：只读英文→漏掉中国真实情况；只读中文→漏掉海外最新动态
