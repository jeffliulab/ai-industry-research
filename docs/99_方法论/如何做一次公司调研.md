# 如何做一次公司调研

> 最后更新：2026-04-22

## 摘要
一次合格的公司调研不是"把维基百科改写一遍"，而是回答三个问题：**这家公司靠什么赚钱？它为什么现在能赚到这些钱？什么情况下它会赚不到？** 下面是我固定使用的八段式结构与每一段的取材方法。

## 一、公司速览
**目标**：30 秒让读者知道"这是什么公司、处于什么阶段"。

**必含字段**：
- 创立时间 / 总部 / 创始团队背景
- 当前员工数量级（"约 XXX 人"，避免精确数字过期快）
- 一句话业务（能被一个非业内人听懂）
- 最新估值或市值 + 最近一轮融资 + 主要投资方

**取材**：Crunchbase / PitchBook 摘要、官网 About 页、最新财报或融资新闻。**不要**用百度百科或维基百科作为唯一来源。

## 二、历史沿革
**目标**：看懂现在的样子是怎么来的。

**方法**：以"关键转折"而非"里程碑清单"组织。每个转折要有**时间 · 事件 · 它改变了什么**三要素。

**常见的关键转折**：
- 换 CEO / 换核心创始人
- 变换主营业务（pivot）
- 关键并购或被并购
- 监管重大事件
- 从融资驱动转向盈利驱动

**取材**：创始人访谈（Podcast、长篇专访）> 企业自传 > 新闻编年史。

## 三、业务与产品
**目标**：分清"收入支柱"和"叙事产品"——它们常常不是同一个东西。

**常用表格**：

| 业务线 | 主要产品 | 目标客户 | 收入占比 | 毛利水平 |
|---|---|---|---|---|
| … | … | … | … | … |

**取材**：财报的 segment 披露 > 招股书的业务章节 > 官网产品页。

!!! tip "分清'故事'与'现金流'"
    很多 AI 公司叙事在 A 产品（面向开发者或消费者）上，但收入在 B 产品（面向企业）上。识别这个错配是调研的关键。

## 四、技术路线
**目标**：说清技术差异化在哪里、是否真的是差异化。

**问题清单**：
1. 底层模型自研还是外采？
2. 如果自研，训练数据来源？算力来源？
3. 如果外采，锁定风险如何？
4. 工程侧有没有"看不见的壁垒"（推理效率、工具链、评测体系）？
5. 有没有专利 / 论文 / 开源作品印证？

**取材**：官方技术博客、论文、GitHub、核心工程师的推特和演讲。警惕 PR 文案。

## 五、商业模式
**目标**：画出一张"钱如何流进来、又如何流出去"的图。

**必答问题**：
- 定价方式（订阅 / 按量 / License / 服务费）
- 单位经济学（LTV / CAC，或毛利率 + 留存率）
- 客户集中度（前 10 大客户占比）
- 获客渠道（Self-serve / 销售团队 / 渠道伙伴）

**取材**：招股书最详细；未上市公司从访谈、离职员工博文、同行对照推测。

## 六、竞争与壁垒
**目标**：不吹不黑地写清"它为什么还活着，以及谁可能把它干掉"。

**三层分析**：
1. **直接对标**：产品功能对比表
2. **替代竞争**：用户不买它会买什么？（有时是 Google 表格+人力）
3. **未来竞争**：上游或下游会不会直接下场？

**护城河检查**：网络效应 / 规模经济 / 品牌 / 切换成本 / 数据 / 监管牌照——挨个打勾。

## 七、关键风险
**目标**：列出"破局点"。每条风险都要具体：**触发条件 + 影响路径 + 观察指标**。

**常见类别**：
- 监管（数据、金融牌照、出口管制）
- 技术（模型架构变迁、开源冲击）
- 商业（单一大客户依赖、被上游涨价）
- 人才（核心 CTO / 首席科学家流失）
- 地缘政治（芯片、市场准入）

## 八、我的判断
**目标**：明确区分"事实复述"和"个人观点"。

格式建议：
> **我的看法**：……（一到两段。）
>
> **我可能错在哪里**：……（诚实地列出反方论据。）

## 九、信息源
**强信号**（尽量用）：
- 最新财报与 10-K / 招股书
- 创始人 / CEO 的长篇访谈（1 小时以上）
- 核心技术博客、论文、GitHub 活动
- 一线员工的公开演讲
- 离职员工的复盘长文

**弱信号**（交叉验证后用）：
- 新闻稿与 PR 文章
- 咨询公司白皮书
- 媒体二手报道

**避免**：
- 只有"据传"的来源
- 明显带节奏的自媒体
- 公众号 / 小红书 / 知乎未署名的"内部人士"

## 一次调研的建议时间盒

| 深度 | 时长 | 产出 |
|---|---|---|
| 快速了解 | 2 小时 | 300 字速写，八段式前三段 |
| 标准调研 | 1-2 天 | 2000-4000 字，八段式完整 |
| 深度调研 | 1-2 周 | 6000+ 字 + 财务模型 + 多人交叉验证 |

本站默认采用**标准调研**深度。
